# 프롬프트 엔지니어링 심화

## 1. Chain-of-Thought (CoT)

### 정의

**Chain-of-Thought (CoT)**는 LLM의 **논리적 추론 과정을 유도하는 기법**입니다.

### Standard Prompting vs CoT Prompting

| 방식 | 특징 | 정답률 |
|------|------|--------|
| **Standard Prompting**| 질문에 바로 답변 | 낮음 |
| **CoT Prompting**| 풀이 과정을 제시하여 단계별 추론 | 높음 |

### CoT의 효과

```
"논리적 추론은 정답으로의 안전한 길" (2022년)
"출력을 길게 하는 방식이 모델의 지능을 높임" (현재)
```

- GPT-5, Claude와 같은 모델의 추론 메타 형성 계기

### 주의사항

<Callout type="warning" title="Over Thinking 문제">
일부 모델은 답을 낼 때 스스로 계속 의심하여 지나친 추론으로 성능이 저하될 수 있습니다. 추론이 무조건 길다고 좋은 것은 아닙니다.
</Callout>

## 2. Zero-shot Chain-of-Thought

### 개념

**Zero-shot-CoT**는 예제 없이도 프롬프트로 논리적 추론을 유도하는 기법입니다.

### 핵심 프롬프트

```
"Let's think step by step." (단계별로 생각해보자)
```

### 효과

- CoT 예제 없이 프롬프트만으로 논리적 추론 유도
- 최근 모델들은 Zero-Shot CoT 방식을 주로 채택

### 언어 모델의 추론을 금지시킨다면?

**질문:**
```
내가 지갑에 2만원이 있었는데, 1만 2천원 배달음식을 주문하고,
2천원 배달팁을 냈어. 이후, 은행에 들러서 5만원권 20장을 인출했어.
내가 가진 현금은 얼마일까?
```

**"풀이 과정을 쓰지 말고, 정답만 출력해줘"**→ 오답

**올바른 풀이 과정:**
1. 초기 현금: 2만원
2. 지출: 1만 2천원 + 2천원 = 1만 4천원
3. 남은 금액: 2만원 - 1만 4천원 = 6천원
4. 인출: 5만원 × 20 = 100만원
5. **최종: 100만 6천원**

<Callout type="info" title="LLM 특성">
LLM은 답을 내는데 필요한 중간 과정 출력이 필요합니다. 물리적·시간적 과정을 거친 뒤 정교해진 답변을 출력합니다.
</Callout>

## 3. Step-Back Prompting

### 정의

**답을 바로 생성하기 전에, 필요한 요소와 구조를 도출**하여 LLM의 한계를 보완하는 기법입니다.

### 배경

LLM은 **전체 구조나 큰 그림을 미리 보는 능력이 부족**하여, 복잡한 문제에서 부정확한 답변 생성 가능

### 사용법 예시

| 상황 | 잘못된 방법 | 올바른 방법 |
|------|-------------|-------------|
| **바이브 코딩**| "코드 짜줘" | "프로그램을 만들고 싶은데, 전체 구획과 구조를 알려줘" |
| **보고서 작성**| "보고서 써줘" | "보고서를 쓸 건데, 전체 구획과 구조를 알려줘" |
| **제안서 평가**| "이 제안서 평가해줘" | "어떤 제안서가 좋을지 평가 기준을 만들어줘" |

### Step-Back Prompting 프로세스

```
Original Question
      ↓
Step 1: Abstraction → Stepback Question
      ↓
Step 2: Reasoning → Stepback Answer
      ↓
Final Answer
```

### 효과

- 논리적 단계를 유도하여 **답변의 일관성, 정확성, 품질 향상**
- **별도 코드 작성 없이 프롬프트 설계**를 통한 답변 생성 준비

## 4. 프롬프트 길이와 비용-속도 관계

### 결과뿐 아니라 비용도 핵심 요소

| 모델 유형 | 출력 길이 증가 시 |
|-----------|-------------------|
| **오픈 LLM**| GPU 소모량 증가 |
| **상용 LLM**| 토큰 당 비용 증가 |

### GPT API 사용 비용 예시

| 모델 | 입력 비용 | 출력 비용 | 특징 |
|------|-----------|-----------|------|
| **GPT-5**| $1.25/100만 토큰 | $10.00/100만 토큰 | 최상의 성능 |
| **GPT-5 mini**| $0.25/100만 토큰 | $2.00/100만 토큰 | 빠르고 저렴 |
| **GPT-5 nano**| $0.05/100만 토큰 | $0.40/100만 토큰 | 요약/분류에 적합 |

### 오픈 LLM 고려사항

- 비용은 들지 않으나 출력 길이가 길어질수록 GPU 소모 증가
- 추론 시간이 길어지면 처리 속도가 느려지고, 성능 저하 위험
- **성능이 떨어지지 않는 설계 필요**

## 5. 프롬프트 작성 Tip

### 영어 vs 한국어 프롬프트

| 언어 | 장점 | 단점 |
|------|------|------|
| **영어**| 토큰 효율성 높음, 안정적 결과 | 작성 시간 소요 |
| **한국어**| 직관적 작성 | 일부 모델에서 성능 저하 |

### 영어 지시의 장점

- 영어 학습량이 **압도적인 대부분 모델**→ 안정적 결과
- 동일 내용 작성 시 **토큰 수 짧음**→ 효율적 처리

### 한국어 모델 선택 기준

<Callout type="info" title="전문 지식보다 한국어 성능이 더 중요">
한국어 미흡 모델은 정확한 답변 생성이 어렵습니다. 한국어 리더보드를 참고하여 개발 시 한국어 성능 지표를 객관적으로 확인하세요.
</Callout>

### 주요 한국어 모델

| 개발사 | 모델 | 특징 |
|--------|------|------|
| **카카오**| Kanana | 한국어 처리 우수 |
| **네이버**| HyperCLOVA | 한국어 특화 |
| **LG**| EXAONE | 한국어 리즈닝 특화 |
| **Upstage**| Solar | DUS 기법 적용 |
| **SKT**| a-dot 4.0 | Qwen 기반 한국어 추가 학습 |

## 핵심 요약

### CoT (Chain-of-Thought)

| 개념 | 설명 |
|------|------|
| **정의**| LLM의 논리적 추론 과정 유도 기법 |
| **핵심 프롬프트**| "Let's think step by step" |
| **효과**| 복잡한 추론 문제 해결력 향상 |

### Step-Back Prompting

| 개념 | 설명 |
|------|------|
| **정의**| 답변 생성 전, 전체 구조와 요소를 먼저 도출 |
| **활용**| 바이브 코딩, 보고서 작성, 제안서 평가 |
| **효과**| 답변의 일관성, 정확성, 품질 향상 |

### 비용-속도 관계

| 요인 | 영향 |
|------|------|
| **프롬프트 길이 ↑**| 비용 ↑, 속도 ↓ |
| **오픈 LLM**| GPU 소모 고려 |
| **상용 LLM**| API 비용 구조 파악 필수 |

### 한국어 모델 선택

- 전문 지식보다 **한국어 성능이 더 중요**
- 호랑이 리더보드 등 객관적 지표 참고
- SKT, KT, LG, Upstage, 네이버, 카카오 등 국내 모델 우수
