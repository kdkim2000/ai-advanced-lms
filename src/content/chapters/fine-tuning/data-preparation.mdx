# 파인 튜닝 데이터 구축하기

## 1. 파인 튜닝 데이터 구축 개요

### 파인 튜닝의 필요성

Fine-tuning은 사전 훈련된 대규모 언어 모델(LLM)을 특정 도메인이나 태스크에 맞게 조정하는 과정입니다.

**핵심 작업**:
- 관련 도메인의 다양한 데이터 크롤링 및 수집
- 데이터 형식 변환 및 전처리
- 품질 검증 및 필터링

### LLM을 활용한 데이터 확장·증강

| 방법 | 설명 |
|------|------|
| **수집 (Collection)** | 원본 문서를 발굴하여 불러오기 |
| **증강 (Augmentation)** | 기존 데이터를 가공하여 확장 |
| **증류 (Distillation)** | 고성능 LLM을 통한 데이터 생성 |

---

## 2. 생성 단계의 주요 고려사항

### 모델별 규정 준수

**OpenAI 정책**:
> "Cannot Use Output to develop models that compete with OpenAI"

- OpenAI의 출력을 사용하여 경쟁 모델 개발 불가
- 파인 튜닝 시 OpenAI API 출력 사용 주의

**Qwen 정책**:
- 학습 사용 시 모델 설명에 명시 필요
- 데이터 출처와 사용 방법 투명하게 공개

### 데이터 품질 관리

**필수 요소**:
1. 모델마다 규정이 다르므로 숙지 필요
2. 생성된 데이터의 정확성 검증
3. 편향(Bias) 및 오류 제거
4. 도메인 특화 정보의 정확성 확보

---

## 3. 어떤 데이터를 준비해야 하는가?

### 문제 해결 중심의 데이터

**핵심 원칙**:
- 풀고자 하는 문제를 충분히 표현할 만한 데이터
- 단순한 지식 주입보다는 **문제 해결 능력 향상**에 초점

### Instruction Tuning의 특성

**한계**:
- 지식을 주입하기 어려움
- 단순 예시 학습만으로는 새로운 문제 해결 능력 부족

```
1번 문제 답변 제시 + 2번 문제 답변 제시
           ↓
3번 문제를 풀 수 있는가?
```

→ 일반화를 위해 다양한 예시를 통한 패턴 학습 필요

---

## 4. Instruction Data 구성 전략

### 기본 구조

```
Instruction (지시문) + Input (입력) → Output (출력)
```

### 일반화를 위한 데이터 설계

**핵심 질문**:
- 일반화 측면에서 어떻게 표준화된 지식을 넣어줄 것인가?
- 충분한 다양성과 깊이를 가진 instruction 데이터 필요

---

## 5. 코퍼스 활용 전략

### 코퍼스의 중요성

**코퍼스란?**
- 다양한 Instruction을 구하기 어려운 상황에서 활용하는 대규모 텍스트 데이터

**활용 방법**:
- 코퍼스를 활용한 QA 세트 생성
- 도메인별 전문 코퍼스 구축

### QA 세트 생성 프로세스

```
1. 코퍼스 수집 및 정제
       ↓
2. 관련 주제에 대한 이해 확장
       ↓
3. QA 쌍 자동 생성
       ↓
4. 품질 검증 및 필터링
```

---

## 6. 좋은 모델 활용과 데이터 품질

### 모델 품질과 데이터 품질의 관계

**핵심 원칙**:
- 좋은 모델을 사용하면 데이터의 **퀄리티가 좋아짐**
- 고성능 모델 활용 시 더 정확하고 자연스러운 데이터 생성 가능

### 지식 증류 (Knowledge Distillation)

```
큰 모델 (Teacher Model)
         ↓ 입력
나온 데이터 (High-Quality Output)
         ↓ 학습
작은 모델 (Student Model)
```

**장점**:
- 비용 효율적이면서도 성능 유지 가능
- 대형 모델의 지식을 소형 모델로 전달

---

## 7. 합성 데이터와 실제 데이터의 결합

### 데이터 균형의 중요성

**핵심 전략**:
- 합성 데이터만 사용하기보다는 **실제 데이터와 결합**
- 합성 데이터의 한계 보완

### 실제 데이터의 가치

| 장점 | 설명 |
|------|------|
| **실제 사용 사례 반영** | 현실적인 시나리오 |
| **자연스러운 표현** | 실제 맥락 포함 |
| **Edge case 포함**| 예상치 못한 상황 |
| **일반화 성능 향상**| 모델의 범용성 증가 |

---

## 8. Fine-tuning 데이터 구축 프로세스

### 전체 프로세스

```
1. 도메인 선정
       ↓
2. 데이터 수집 (크롤링, 코퍼스)
       ↓
3. 데이터 전처리 및 형식 변환
       ↓
4. LLM 활용 데이터 증강
       ↓
5. Instruction Data 생성
       ↓
6. 품질 검증 및 필터링
       ↓
7. 실제 데이터와 합성 데이터 결합
       ↓
8. Fine-tuning 실행
```

### 데이터 구축 시 체크리스트

- [ ] 모델 사용 정책 확인 (OpenAI, Qwen 등)
- [ ] 데이터 품질 관리 체계 수립
- [ ] 충분한 다양성 확보
- [ ] 실제 데이터와 합성 데이터의 적절한 비율
- [ ] 도메인 전문가 검토
- [ ] 편향 및 오류 제거
- [ ] 저작권 및 개인정보 보호 준수

---

## 9. 파인 튜닝 데이터 구축의 3대 원칙

### 1. 품질 우선
좋은 모델로 생성된 고품질 데이터 활용

### 2. 규정 준수
각 모델의 사용 정책 철저히 준수

### 3. 균형
합성 데이터와 실제 데이터의 적절한 조합

---

## 10. 실전 적용 가이드

### 초보자를 위한 시작 가이드

**Step 1: 도메인 선정**
- 명확한 목표 설정
- 해결하고자 하는 문제 정의

**Step 2: 데이터 수집**
- 관련 도메인의 공개 데이터셋 활용
- 크롤링을 통한 추가 데이터 수집

**Step 3: LLM 활용**
- 고성능 LLM을 활용한 데이터 증강
- 품질 검증 프로세스 구축

### 주의사항

**반드시 피해야 할 사항**:
1. 모델 사용 정책 위반
2. 품질 검증 없는 대량 데이터 생성
3. 합성 데이터에만 의존
4. 도메인 특성 무시

**권장사항**:
1. 소량의 고품질 데이터로 시작
2. 점진적인 데이터 확장
3. 지속적인 품질 모니터링
4. 실제 사용 사례 기반 개선

## 핵심 요약

### 성공적인 Fine-tuning을 위한 핵심 요소

| 요소 | 설명 |
|------|------|
| **도메인 이해** | 풀고자 하는 문제에 대한 깊은 이해 |
| **데이터 다양성** | 충분한 variation을 가진 instruction 데이터 |
| **품질 관리** | 지속적인 검증 및 개선 프로세스 |
| **지식 증류** | 큰 모델의 지식을 작은 모델로 효율적 전달 |
| **실제 데이터** | 합성 데이터만으로는 부족, 실제 사례 필수 |

### 핵심 키워드

- **Fine-tuning**: 사전 학습 모델의 추가 학습
- **Instruction Tuning**: 지시사항 기반 학습
- **Knowledge Distillation**: 지식 증류
- **Data Augmentation**: 데이터 증강
- **Corpus**: 대규모 텍스트 데이터
- **Synthetic Data**: 합성 데이터
- **Quality Control**: 품질 관리
