# Instruction Tuning 실전

## 1. Instruction Tuning의 개념

### 정의

**Instruction Tuning (IT)** 은 사전 학습된 언어 모델을 특정 명령어(instruction)에 따라 응답하도록 미세 조정하는 기법입니다.

### 목적

- 모델이 다양한 사용자 명령을 이해하고 적절히 응답하도록 학습
- 질의응답(QA) 형식의 입력-출력 템플릿 학습
- 일반 대화부터 전문 영역까지 다양한 태스크 수행 능력 향상

### 주요 특징

| 특징 | 설명 |
|------|------|
| **입력-출력 쌍 학습**| 명령어와 적절한 응답을 학습 |
| ** 제로샷/퓨샷 성능 향상**| 새로운 태스크에 대한 일반화 능력 개선 |
| ** 사용자 의도 파악**| 자연어 명령의 의미를 정확히 이해 |

---

## 2. CPT 모델의 질의응답 학습

### CPT (Continual Pre-Training)

- ** 개념**: 기존 사전 학습된 모델을 특정 도메인 데이터로 추가 학습
- ** 목적**: 일반 지식을 유지하면서 특정 영역의 전문성 향상

### IT를 통한 CPT 모델의 질의응답 학습

** 학습 데이터의 중요성**:
```
모델의 다양한 능력을 키우고 망각을 방지하기 위해
다양한 데이터를 섞어서 학습하는 것이 필요
```

**CPT 모델의 현재 상태**:
- 질의응답(QA)을 모르는 상태
- 단순한 QA 데이터로는 템플릿 이해 불가

** 해결 방법**:
```
입력-출력 템플릿 학습을 위해
일반 QA 데이터 추가 필요
```

## 3. Instruction Tuning을 위한 데이터 전략

### 데이터 구성 원칙

** 다양성 확보**:
- 목적별 데이터: 질의응답, 요약, 번역, 분석 등
- 도메인별 데이터: 일반 지식, 전문 영역, 업무 관련 등
- 난이도별 데이터: 단순 질문부터 복잡한 추론까지

** 품질 관리**:
- 명확한 지시문: 모호하지 않은 명령어 작성
- 적절한 응답: 지시문에 정확히 부합하는 답변
- 일관성 유지: 유사한 질문에 대한 일관된 응답 스타일

### 데이터 혼합 전략

**Instruction Tuning 데이터셋 구성**:

- ** 일반 QA 데이터 (템플릿 학습)**
  - 기본적인 입력-출력 구조 학습
  - 다양한 질문 유형 경험

- ** 도메인 특화 데이터**
  - 목표 태스크 관련 데이터
  - 전문 용어 및 맥락 학습

- ** 복잡한 추론 데이터**
  - 다단계 사고 과정 학습
  - 논리적 추론 능력 향상

### 데이터 형식

** 기본 구조**:
```json
{
  "instruction": "사용자의 명령어 또는 질문",
  "input": "추가 컨텍스트 (선택사항)",
  "output": "모델의 기대 응답"
}
```

** 예시 1: 간단한 QA**
```json
{
  "instruction": "대한민국의 수도는 어디인가요?",
  "input": "",
  "output": "대한민국의 수도는 서울입니다."
}
```

** 예시 2: 컨텍스트 기반 질문**
```json
{
  "instruction": "다음 문장을 요약해주세요.",
  "input": "인공지능은 인간의 학습능력, 추론능력, 지각능력을 인공적으로 구현한 컴퓨터 프로그램입니다.",
  "output": "인공지능은 인간의 지능적 능력을 컴퓨터로 구현한 시스템입니다."
}
```

## 4. 실무 적용 고려사항

### 컴퓨팅 자원

** 대량 데이터 처리 요구사항**:
- 무료 계정의 GPU 사용 시간 및 메모리 제한
- 대규모 모델 학습 시 충분한 자원 필요
- 장시간 학습 작업의 안정성 확보

** 리소스 선택 가이드**:

| 데이터 규모 | 권장 환경 | GPU |
|------------|----------|-----|
| 소규모 (10K 미만) | Colab 무료 | T4 |
| 중규모 (10K~100K) | Colab Pro | V100 |
| 대규모 (100K 이상) | Colab Pro+ 또는 전용 서버 | A100 |

### 학습 파라미터 설정

```python
# 일반적인 Instruction Tuning 설정
training_args = {
    "learning_rate": 2e-5,        # 학습률
    "num_train_epochs": 3,         # 에폭 수
    "per_device_train_batch_size": 4,  # 배치 크기
    "gradient_accumulation_steps": 4,  # 그래디언트 누적
    "warmup_steps": 100,          # 워밍업 스텝
    "weight_decay": 0.01,         # 가중치 감쇠
    "logging_steps": 10           # 로깅 주기
}
```

### 효율적인 학습 기법

**LoRA (Low-Rank Adaptation)**:
- 전체 파라미터의 일부만 학습하여 메모리 절약
- 대규모 모델을 제한된 자원에서 학습 시 유용

**QLoRA (Quantized LoRA)**:
- 양자화를 통해 추가 메모리 절약
- GPU 메모리가 매우 제한적인 환경

## 5. Instruction Tuning vs 다른 미세조정 방법

### 비교표

| 특성 | Instruction Tuning | SFT | RLHF |
|-----|-------------------|-----|------|
| ** 데이터 형식** | Instruction-Output 쌍 | Task-specific 데이터 | Preference 데이터 |
| ** 학습 목표** | 다양한 명령 수행 | 특정 태스크 최적화 | 인간 선호도 반영 |
| ** 데이터 준비** | 상대적으로 쉬움 | 태스크별 레이블링 | 선호도 레이블링 필요 |
| ** 일반화 능력** | 높음 | 중간 | 높음 |
| ** 학습 난이도** | 중간 | 낮음 | 높음 |

### 각 방법의 적용 시나리오

**Instruction Tuning**:
- 범용 챗봇 개발
- 다양한 태스크 수행 모델
- 제로샷 성능 향상 필요 시

**SFT (Supervised Fine-Tuning)**:
- 특정 분류 태스크
- 명확한 정답이 있는 작업
- 도메인 특화 모델

**RLHF (Reinforcement Learning from Human Feedback)**:
- 인간의 선호도 반영 필요 시
- 안전성과 윤리성 강화
- 대화 품질 개선

## 6. 실전 예제: Instruction Tuning 파이프라인

### 데이터 준비

```python
# 데이터 로드
import json

def load_instruction_data(file_path):
    with open(file_path, 'r', encoding='utf-8') as f:
        data = json.load(f)
    return data

# 데이터 전처리
def format_instruction(example):
    """Instruction 형식으로 변환"""
    if example.get("input"):
        prompt = f"### Instruction:\n{example['instruction']}\n\n### Input:\n{example['input']}\n\n### Response:\n"
    else:
        prompt = f"### Instruction:\n{example['instruction']}\n\n### Response:\n"

    return {
        "prompt": prompt,
        "completion": example["output"]
    }
```

### 모델 로드 및 설정

```python
from transformers import AutoModelForCausalLM, AutoTokenizer
from peft import LoraConfig, get_peft_model

# 기본 모델 로드
model_name = "your-base-model"
model = AutoModelForCausalLM.from_pretrained(model_name)
tokenizer = AutoTokenizer.from_pretrained(model_name)

# LoRA 설정
lora_config = LoraConfig(
    r=8,
    lora_alpha=32,
    target_modules=["q_proj", "v_proj"],
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM"
)

# LoRA 모델 생성
model = get_peft_model(model, lora_config)
```

### 학습 실행

```python
from transformers import Trainer, TrainingArguments

# 학습 인자 설정
training_args = TrainingArguments(
    output_dir="./instruction_tuned_model",
    num_train_epochs=3,
    per_device_train_batch_size=4,
    gradient_accumulation_steps=4,
    learning_rate=2e-5,
    fp16=True,
    logging_steps=10,
    save_steps=100,
    evaluation_strategy="steps",
    eval_steps=100
)

# Trainer 초기화
trainer = Trainer(
```

## 7. 평가 및 검증

### 평가 메트릭

** 자동 평가**:
- **BLEU**: 생성 텍스트와 참조 텍스트의 n-gram 일치도
- **ROUGE**: 요약 품질 평가
- **Perplexity**: 언어 모델의 예측 정확도

** 인간 평가**:
- ** 유용성 (Helpfulness)**: 답변이 질문에 도움이 되는가?
- ** 정확성 (Accuracy)**: 사실적으로 정확한가?
- ** 일관성 (Coherence)**: 논리적으로 일관된가?
- ** 유창성 (Fluency)**: 자연스러운 언어인가?

### 검증 방법

```python
def evaluate_model(model, tokenizer, test_prompts):
    results = []

    for prompt in test_prompts:
        inputs = tokenizer(prompt, return_tensors="pt")
        outputs = model.generate(**inputs, max_length=256)
        response = tokenizer.decode(outputs[0], skip_special_tokens=True)

        results.append({
            "prompt": prompt,
            "response": response
        })

    return results
```

## 8. 일반적인 문제 및 해결 방법

### Catastrophic Forgetting (파국적 망각)

** 문제**: 새로운 태스크 학습 시 이전 지식 손실

** 해결 방법**:
1. ** 데이터 혼합**: 일반 데이터와 특화 데이터 함께 사용
2. **EWC**: 중요한 가중치 보호
3. ** 적절한 학습률**: 너무 높은 학습률 피하기

### Overfitting (과적합)

** 문제**: 학습 데이터에만 과도하게 최적화

** 해결 방법**:
1. ** 데이터 증강**: 다양한 표현 방식 추가
2. ** 드롭아웃**: 정규화 기법 적용
3. **Early Stopping**: 검증 성능 기준 조기 종료

### 리소스 부족

** 문제**: GPU 메모리 부족, 학습 시간 과다

** 해결 방법**:
1. **LoRA/QLoRA**: 효율적인 미세조정 기법
2. **Gradient Checkpointing**: 메모리 절약
3. **Mixed Precision Training**: FP16/BF16 사용
4. ** 배치 크기 조정**: Gradient Accumulation 활용

## 핵심 요약

### Instruction Tuning의 핵심

| 항목 | 설명 |
|------|------|
| ** 목적** | 사전 학습 모델이 다양한 명령어를 이해하고 수행 |
| ** 데이터** | Instruction-Input-Output 형식의 다양한 데이터 |
| ** 방법** | 일반 QA 데이터로 템플릿 학습 후 특화 데이터로 강화 |
| ** 자원** | 대규모 학습 시 충분한 컴퓨팅 자원 필요 |

### CPT 모델 학습의 핵심

- 생성한 데이터만으로는 불충분
- 일반 QA 데이터로 입력-출력 템플릿 먼저 학습
- 다양한 데이터 혼합으로 망각 방지

### 실무 적용 팁

- LoRA/QLoRA로 효율적 학습
- 충분한 평가 및 검증
- 점진적 개선과 모니터링
