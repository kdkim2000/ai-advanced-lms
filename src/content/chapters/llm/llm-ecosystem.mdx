# LLM 생태계와 주요 모델들

## 1. LLM 성능 현황

### 주요 모델 순위

**현재 시장 상황**:
- GPT 계열 모델이 성능 1위 유지
- Gemini, Claude, Grok 등도 Top 10 안에 위치

| Model | Creator | Context Window | 특징 |
|-------|---------|----------------|------|
| **GPT-5** | OpenAI | 400k | 최고 성능 |
| **Grok 4** | xAI | 256k | 추론 특화 |
| **Gemini 2.5 Pro** | Google | 1M | 긴 컨텍스트 |
| **Claude 4.1 Opus** | Anthropic | 200k | Thinking 모델 |
| **Qwen3 235B** | Alibaba | 256k | 오픈 모델 최강 |

### 모델 구분

| 구분 | 상용 모델 (Proprietary) | 오픈 모델 (Open) |
|------|------------------------|------------------|
| ** 공개 여부** | 기술 비공개 | 파라미터 공개 |
| ** 사용 방법** | API 통해서만 사용 | GPU로 직접 실행 |
| ** 예시** | GPT, Claude, Gemini | Llama, Mistral, Qwen |

## 2. 오픈 모델의 장단점

### 장점

| 항목 | 설명 |
|------|------|
| ** 프라이버시** | 민감 데이터 보안 처리 가능 (금융·기관 활용) |
| ** 비용** | API 호출 비용 절약, 예측 가능한 비용 |
| ** 파인튜닝** | 특수 목적/데이터에 맞춤형 모델 제작 가능 |
| ** 내부망** | 온디바이스·내부망 환경 최적화 가능 |

### 단점

- GPU, 계산량, 비용이 많이 듦
- 상위권 성능은 여전히 상용 모델이 우위

- ** 합성 데이터를 많이 활용**

### 특징

|------|------|
| 사이즈 대비 성능 우수 | 한국어 성능 부족 |
| AWS에서 바로 사용 가능 | 토큰 효율 낮음 |

## 5. Alibaba Qwen Series

### 현존 최고 다국어 처리 모델

- 초기부터 ** 멀티링구얼 모델** 표방
- 언어별 최적화된 토크나이저 설계
- SLM부터 대형 LLM까지 폭넓게 공개

### Qwen3 모델 크기와 구조

| 모델 | 크기 | 특징 |
|------|------|------|
| Dense | 0.6B ~ 32B | 일반 모델 |
| MoE | 30B (활성화 3B) | 속도는 3B, 성능은 30B |

** 레이어 변화**: 28개 → 36개 → 40개 → 64개...
- 작은 모델: 레이어 수 적음
- 큰 모델: 레이어 수 많음 → 성능 향상

## 9. 파라미터와 GPU 메모리

### 파라미터 수 개념

- **B 단위**: 7B = 70억 개, 22B = 220억 개
- 초대형 모델: GPT-4 (1.76T), Claude 3 Opus (2T)

### GPU 메모리 계산법

**16bit 모델 기준**:
```
7B 모델 = 70억 × 2바이트 = 14GB
```

| bit | 계산 공식 |
|-----|-----------|
| **16bit** | 파라미터 수 × 2 = GB |
| **32bit** | 파라미터 수 × 4 = GB |

### 실제 사용 시 필요 메모리

| 용도 | 필요 메모리 |
|------|-------------|
| ** 안정적 사용** | 기본 × 1.5배 |
| ** 학습** | 기본 × 10배 이상 |

## 10. 양자화 (Quantization)

### 양자화란?

**Weight 값을 줄여 모델 크기를 작게 만드는 과정**

### INT8 양자화 원리

- 최솟값과 최댓값 기준으로 255 단계 분할
- 가중치 분포(0~1) → 0~255 값으로 매핑

### 양자화의 장단점

| 장점 | 단점 |
|------|------|
| 모델 크기 감소 | 성능 저하 |
| GPU 메모리 절감 | 답변 결과 변경 가능 |

---

## 11. 개선된 양자화 기법

### QLoRA: 분위수 기반

- 등 길이 대신 ** 분위수(Quantile) 기준** 분할
- 원래 값 보존률 높음, 아웃라이어에도 강건

### GPTQ

- ** 파라미터 중요도 기반 차등 양자화**
- 중요한 파라미터 → 적게 양자화
- 덜 중요한 파라미터 → 크게 양자화

### AWQ

- 레이어 단위로 ** 입력 활성화 값** 확인
- 가중치의 활성화 정도 반영해 양자화

### GGUF/GGML

- **CPU 추론 최적화 포맷**
- C++ 기반 (라마 CPP)
- 전체 모델 단일 파일로 저장 → 배포 간편
- Ollama 활용 시 손쉬운 서빙

### QAT (Quantization Aware Training)

- ** 학습 단계에서 양자화 고려**
- 중간중간 양자화 적용/해제 반복 학습
- Google Gemma3에 적용

---

## 12. 큰 모델 양자화 vs 작은 모델

### 일반적으로 큰 모델 양자화가 유리

** 이유**:
1. 레이어 개수가 많음
2. 차원이 큼
3. 컨텍스트 길이가 김
4. 많은 데이터를 학습함
