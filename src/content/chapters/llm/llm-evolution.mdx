# LLM의 발전 과정

## 1. Transformer 기반 사전 학습 모델

### BERT와 GPT의 등장 (2018)

**Transformer의 성공 이후 두 가지 방향으로 발전**

| 모델 | 구조 | 역할 |
|------|------|------|
| **BERT**| Encoder 활용 | 문장 의미 파악, 벡터 출력 |
| **GPT**| Decoder 활용 | 텍스트 생성 |

---

## 2. GPT부터 GPT-3까지

### GPT-1 (2018): 0.1B 파라미터

**파라미터가 1억 개인 Transformer Decoder 모델**

- 특정 도메인에 적용하기 위해 **Fine-tuning 필요**
- 예시: 리뷰 데이터 + 레이블 추가 학습 후 긍정/부정 분류

### GPT-2 (2019): 1.5B 파라미터

**모델 크기를 늘리고 웹 텍스트로 학습**

- 긴글 뒤에 **TL;DR**(Too Long; Didn't Read)을 붙이면 요약 생성
- 간단한 요약, 번역 등의 작업을 언어로 전달 가능

### GPT-3 (2020): 175B 파라미터

**대량의 데이터를 바탕으로 높은 수준의 지식과 언어 능력을 갖춘 LLM의 시초**

#### 주요 능력 발견

**1. In-Context Learning**
- 입력 데이터를 주고 관련 내용을 붙이면 해당 데이터를 바탕으로 답변
- 예시: 메뉴판 가격과 함께 'A 메뉴의 가격은?'이라고 입력 → '3만 7천원' 답변

**2. Few-Shot Learning**
- 최소한의 데이터(예시)를 통해 패턴을 파악하여 답변
- 예시:
  - 긍정/부정 리뷰 예시 제공 후 새 리뷰 분류
  - '쌍둥이:Twin, 독수리:Eagle, 사자:' → '사자:Lion'

**3. 질의응답 능력 부족**
- '오늘이 무슨 요일입니까?' 입력 시 질문 이어쓰기/답변 혼동
- 일관된 답변 어려움 → GPT-3.5에서 개선

---

## 3. GPT-3.5: ChatGPT의 시작

### GPT-3.5 (2022): 175B 파라미터

**GPT-3을 개량해서 만든 ChatGPT 모델의 시초**

#### 핵심 개선사항
- 질의 응답 포맷 학습
- 강화 학습을 이용해 **인간의 선호도 반영**

#### 학습 방법: SFT + RLHF

```
Step 1: SFT → Step 2: Reward Model 학습 → Step 3: RLHF
```

---

## 4. SFT (Supervised Fine Tuning)

### 정답이 있는 Fine-tuning

**질문 + 답변 세트 포맷을 만들어 학습**

```
User: 6살 아이한테 달 착륙에 대해 설명해주세요.
답변: 달에는 우주선을 타고 갔어요...
```

### SFT의 특징

- 질문에 답을 해야 할 **타이밍**을 템플릿으로 학습
- 답변의 **일관성**향상
- 나쁜 질문에도 적절히 대응 가능

<Callout type="warning" title="SFT만으로는 부족">
인간의 선호도에 맞는 답변을 하기 위한 추가 학습이 필요합니다.
</Callout>

---

## 5. Reward Model 학습

### 좋은 답변을 판단하는 모델 학습

**사람에게 질문에 대한 좋은 답변이 무엇인지 라벨링**

#### 예시 프로세스

질문에 대해 4개의 답변 생성:
- A: 중력을 설명해 주세요.
- B: 달은 지구의 위성입니다.
- C: 달에 가서 사진을 찍었습니다.
- D: 사람들이 달에 착륙했습니다.

**사람 평가**: A → B → D → C 순서로 순위 매김

### Reward Model 학습

- 라벨링 정보를 토대로 Reward Model 학습
- 0 (나쁜 답변) vs 1 (좋은 답변)
- **바르고 유익한 말일수록 높은 점수**

---

## 6. RLHF (Reinforcement Learning with Human Feedback)

### LLM의 출력을 인간 선호도에 맞게 조정

**프로세스**:
1. 새로운 프롬프트 샘플링
2. Policy(모델)가 출력 생성
3. Reward Model이 인간 선호도 반영하여 평가
4. Reward 점수가 높은 방향으로 다시 학습

<Callout type="info" title="RLHF의 역할">
LLM의 출력을 인간 선호도에 맞게 바꿔주는 강화학습입니다.
</Callout>

### GPT-3.5 전체 학습 흐름

```
[Step 1: SFT] → [Step 2: Reward Model] → [Step 3: RLHF]
```

**부작용**: Reward 학습으로 인해 객관성을 잃고 아첨하는 경향 발생

## 7. GPT-4: MoE 기법 도입

### GPT-4 (2023): 1760B 파라미터

**N개의 모델 혼합: Mixture of Experts (MoE) 기법**

### MoE 구조

**Self-Attention 공유 + 여러 개의 MLP 중 일부만 활성화**

#### Router의 역할
- 여러 개의 FFN 중에서 문제를 풀 FFN을 결정하는 모듈
- 예시: 이탈리아어로 된 수학 문제 → 3번과 7번 FFN 활성화

### MoE의 장점

| 장점 | 설명 |
|------|------|
| **효율성**| 모델은 크게, 실제 계산량은 줄임 |
| **속도**| 출력이 빨라짐 |
| **확장성**| 이후 DeepSeek V2 등에서 개선된 구조로 계속 활용 |

## 8. GPT-4o: 멀티모달 통합

### Early Fusion 방식

**텍스트/이미지/오디오/비디오를 멀티미디어 토큰으로 변환 → 초기에 융합하여 학습**

### GPT-4 vs GPT-4o

| 구분 | GPT-4 | GPT-4o |
|------|-------|--------|
| **이미지 처리**| Vision Encoder + Projection layer | 초기 융합 (Early Fusion) |
| **입출력**| 텍스트 중심 | Any-to-Any (텍스트/오디오/이미지) |
| **토크나이저**| 기존 | 개선 (한국어 등 다국어 처리 속도 증가) |

---

## 9. 추론 전문 모델: o 시리즈

### OpenAI o 시리즈

**질문이 주어지면 긴 추론을 통해 답을 생성하는 모델**

- 추론을 통해 어려운 문제도 잘 해결
- 수학, 코딩 등 복잡한 사고가 필요한 작업에 특화

---

## 10. GPT-5

### 가장 최신 모델

**문제가 주어지면 Thinking을 많이 한 다음 답변**

- 현존하는 모델 중 모든 분야에서 **1등 성능**달성

---

## 11. Decoder 기반 모델이 분류도 잘 하는 이유

### 이론 vs 실제

**이론적으로는 Encoder 모델의 문장 이해력이 더 뛰어남**
- Encoder: 양방향 맥락 참고
- Decoder: 단방향 맥락 참고

**실제로는 Decoder 모델도 높은 이해력**
- 충분한 파라미터 + 충분한 데이터 학습
- 최근 Decoder 기반 모델은 다양한 언어 Task에 모두 활용 가능

## 핵심 요약

### GPT 발전 과정

| 모델 | 파라미터 | 핵심 특징 |
|------|----------|-----------|
| **GPT-1**| 0.1B | Fine-tuning 필요 |
| **GPT-2**| 1.5B | TL;DR 요약 기능 |
| **GPT-3**| 175B | In-Context Learning, Few-Shot Learning |
| **GPT-3.5**| 175B | SFT + RLHF → ChatGPT |
| **GPT-4**| 1760B | MoE 기법 |
| **GPT-4o**| - | Multimodal Early Fusion |
| **GPT-5**| - | 최고 성능, 깊은 추론 |

### 학습 방법

| 방법 | 설명 |
|------|------|
| **SFT**| 질문+답변 포맷 학습 |
| **Reward Model**| 좋은 답변 판단 학습 |
| **RLHF**| 인간 선호도 반영 강화학습 |

### GPT-3 핵심 능력

1. **In-Context Learning**: 입력 데이터 기반 답변
2. **Few-Shot Learning**: 최소 예시로 패턴 파악
3. 질의응답 능력 부족 → GPT-3.5에서 개선
