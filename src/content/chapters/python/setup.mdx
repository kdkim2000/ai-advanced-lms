# Python 개발환경 설정하기

## 1. Python이란?

Python은 **AI와 머신러닝 개발에 가장 많이 쓰이는 프로그래밍 언어** 입니다.

### Python의 특징

| 특징 | 설명 |
|------|------|
| **간결한 문법**| 문법이 간결하고 배우기 쉬움 |
| **풍부한 생태계**| 방대한 라이브러리와 커뮤니티 지원 |
| **범용성**| 웹 개발, 데이터 분석, AI/ML, 자동화 등 다양한 분야에 활용 |
| **LLM 친화적**| 특히 LLM(대규모 언어 모델)에서 가장 많은 활용성을 보임 |

## 2. Python & VS Code 설치

### Python 설치

- **다운로드**: [python.org](https://python.org)에서 최신 버전 다운로드
- **중요**: 설치 시 **"Add python.exe to PATH"** 옵션 반드시 체크!

```bash
# 설치 확인
python --version
```

<Callout type="warning" title="PATH 설정 필수">
PATH에 Python을 추가하지 않으면 터미널에서 python 명령어를 사용할 수 없습니다.
</Callout>

### VS Code 설치

- **다운로드**: [code.visualstudio.com](https://code.visualstudio.com)에서 무료 다운로드
- 오픈 소스 AI 코드 에디터

## 3. Python 실행하기

### 터미널에서 실행

```bash
# 버전 확인
python --version

# Python 인터프리터 실행 (대화형 모드)
python

# 코드 실행
>>> print("Hello, Python!")
Hello, Python!
>>> exit()  # 종료
```

### VS Code에서 실행

- 새 파일 작성 후 **Shift + F5** 또는 터미널에서 **python 파일명.py** 실행
- Python 인터프리터 선택: **Ctrl+Shift+P**→ "Python: Select Interpreter"

## 4. 가상 환경 (Virtual Environment)

### 가상 환경이 필요한 이유

- 별도의 라이브러리 설치 시 **버전이 다르면 실행이 안 되거나 충돌** 발생 가능
- **프로젝트별 독립적인 가상 환경** 을 만들어서 개발

<Callout type="info" title="가상환경 사용 이유">
프로젝트 A는 numpy 1.20이 필요하고, 프로젝트 B는 numpy 2.0이 필요할 때 가상환경으로 분리하면 충돌 없이 개발할 수 있습니다.
</Callout>

### 가상 환경 생성 및 활성화

```bash
# 가상 환경 생성
python -m venv myenv

# 활성화 (Windows)
myenv\Scripts\activate

# 활성화 (Mac/Linux)
source myenv/bin/activate

# 비활성화
deactivate
```

## 5. 패키지 관리 (pip)

```bash
# 패키지 설치
pip install numpy pandas

# 특정 버전 설치
pip install numpy==1.24.0

# 설치된 패키지 목록
pip list

# requirements.txt로 내보내기/설치
pip freeze > requirements.txt
pip install -r requirements.txt
```

## 6. .py 파일과 .ipynb 파일

| 구분 | .py (Python Script) | .ipynb (Jupyter Notebook) |
|------|---------------------|---------------------------|
| **형식**| 일반 Python 코드 파일 | JSON 형식으로 저장된 노트북 파일 |
| **특징**| 실제 개발에 적합 | 코드 + 결과 + 문서(Markdown) 함께 관리 |
| **용도**| 프로덕션 배포, 스크립트 실행 | 데이터 분석, 교육, 실험 |
